# Plugin sources — LSP LUT Generator

- **`core/LSPLutGeneratorPlugin.cpp`** — **`ImageEffect`** instance: **`render`**, **`changedParam`**, factory registration.
- **`core/LSPLutGeneratorDescribe.cpp`** — Clips and parameters (operation mode, LUT size, export button).
- **`core/LSPLutGeneratorProcessor.cpp`** — **`ImageProcessor`**: Generate LUT Table mode fills balanced-grid **LUT table**; Analyze mode copies source → output. **CPU:** **`getPixelAddress`** per pixel. **Metal (host-enabled):** **`processImagesMetal`** + compute kernels in **`metal/LSPLutGeneratorMetal.metal`**.
- **`core/LSPLutGeneratorPattern.cpp`** — Strip sampling math for the **LUT table** grid layout.
- **`core/LSPLutGeneratorCube.cpp`** — Binning + **`.cube`** writer; shrink with box average when **`n_src % n_dst == 0`**, else trilinear. On macOS, **`metal/LSPLutGeneratorMetalCube.mm`** tries Metal first for analyze binning and cube downsample/resample, then CPU.
- **`macos/LSPLutGeneratorDialogs.mm`** — Save **`.cube`** via **NSSavePanel** on main thread.
- **`metal/`** — GPU path only: **`LSPLutGeneratorMetal.mm`**, **`LSPLutGeneratorMetal.metal`** (compute kernels), **`LSPLutGeneratorMetalBridge.h`** (C bridge from **`core/`**). The **`.metal`** file is copied into the bundle **`Resources`** and compiled at load with **`newLibraryWithSource:`** (no separate Apple **`metal`** CLI required for **`make`**).
- **`core/LSPLutGeneratorConstants.h`** — **`kPluginIdentifier`** via **`version_gen.h`**.

**`version_gen.h`** is generated by **`make`** from the first line of **`VERSION`** at the repo root (**`MAJ.MIN.PATCH`** plus optional label, e.g. **`1.0.2 beta`**); do not edit by hand. To bump the patch and keep the label, run **`tools/archive_version.sh`** from the repo root before **`make`**.
